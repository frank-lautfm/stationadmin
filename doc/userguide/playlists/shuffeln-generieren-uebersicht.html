Übersicht - Playlists shuffeln und generieren

Selbstverständlich kannst Du die Titelreihenfolge innerhalb einer Playlist selbst festlegen. Um ein abwechslungsreiches Programm zu gestalten wird man aber normalerweise zumindest für einen Teil der Playlists keine feste Reihenfolge vorgeben wollen.

Station Admin bietet mehrere Möglichkeiten wie die Titelreihenfolge automatisch festgelegt werden kann:

  • auf dem Server shuffeln - hier werden auf dem Server die Titel der Playlist oder eine hinreichende lange Auswahl aus den Titeln in eine zufällige Reihenfolge gebracht. Dazu gibt es mehrere mögliche Verfahren.
  • durch Station Admin shuffeln - hier werden die Titel einer Playlist lokal durch Station Admin in eine zufällige Reihenfolge gebracht und abgespeichert. Der Server spielt die Playlist dann so ab wie sie abgespeichert wurde.
  • durch Station Admin generieren - hier wird die Playlist auf Basis von vorgegebenen Regeln lokal durch Station Admin neu befülltt und abgespeichert. Der Server spielt die Playlist dann so ab wie sie abgespeichert wurde.

Generell gilt: Wird durch Station Admin lokal geshuffelt oder generiert muss Station Admin laufen, selbst wenn das durch Anlegen einer Aufgabe automatisiert abläuft - das ist der große Nachteil. Der Vorteil besteht darin dass das lokale Shuffeln oder Generieren mehr Gestaltungsmöglichkeiten wie etwa das regelbasierte Einbinden von Jingles bietet.

Für das Shuffeln auf dem Server stehen folgende Verfahren zu Auswahl:

  • laut.fm - einfach - einfaches Umsortieren der Titel, nicht weiter konfigurierbar
  • laut.fm - Tag-Pattern - Titelabfolge kann mittels Tags kontrolliert werden ("Stundenuhr")
  • Station Admin - Shuffle - Umsortierung mit vielen Konfigurationsmöglichkeiten
  • Station Admin - Teilplaylist auswählen - zerlegt eine Playlist mit vordefinierter Titelreihenfolge in Teile und wählt einen davon aus. Hierbei werden keine Titel umsortiert - Ziel ist es vielmehr über vorgefertigte Teilplaylists zu rotieren.
  • Station Admin - Originalreihenfolge mit Fortsetzung - spielt die Titel in der Originalreihenfolge ab, setzt die Playlist nach einer Unterbrechung aber nach dem zuletzt gespielten Titel fort insofern dieser innerhalb der letzten 24 Stunden lief
