Analyse von Logfiles

Quelle

Auf stationadmin.emjoy.net werden zentral alle gespielten Titel aller Stations sowie die Hörerzahlen im 10-Minuten-Takt mitprotokolliert. Station Admin kann diese Daten für die eigene Station herunterladen und auswerten. Es gibt derzeit keine Möglichkeit diese Daten direkt von laut.fm zu bekommen.

Einverständnis zum Download

Bevor Station Admin das erste Mal eine Verbindung zu stationadmin.emjoy.net herstellt wirst Du gefragt ob Du damit einverstanden bist - das geschieht wenn Du eine der Funktionen im Analyse-Menü das erste mal aufrufst. Es werden selbstverständlich nur die absolut notwendigen Daten - Stationsname und das Datum für das die Daten abgefragt werden - übertragen.
Du kannst Dein Einverständnis jederzeit unter Extras/Einstellungen auf dem Tab Sonstiges widerrufen. Die Analyse-Funktionen sind dann nicht mehr nutzbar.
Du kannst weiterhin Station Admin erlauben die Daten automatisch beim Programmstart herunterzuladen. Da nur die Daten der letzten 7 Tage vom Server abgerufen werden können ist dies sinnvoll um keine Lücken in den lokal verfügbaren Logs enststehen zu lassen.

Was genau wird heruntergeladen?

Die Logs für die gespielten Titel und die Hörerzahlen werden auf stationadmin.emjoy.net tageweise zur Verfügung gestellt. Die Daten für die vergangenen Tage werden von Station Admin nur einmal abgerufen und lokal abgespeichert. Die Daten für den aktuellen Tag werden nicht lokal abgelegt da sie ja noch nicht vollständig sind - diese werden jedes Mal wenn Station Admin diese benötigt erneut vom Server abgerufen.

Die Daten für die Hörerzahlen beinhalten pro Eintrag die Uhrzeit sowie die Zahl der Hörer zu diesem Zeitpunkt. Diese Zahlen stammen von http://api.laut.fm/listeners.
Die Daten für die gespielten Titel beinhalten pro Titel die Startzeit sowie die interne Titel-ID. Diese Daten wurden über den Abruf von http://api.laut.fm/song_change.stream.json aufgezeichnet.

Vollständigkeit

Das Logging der Daten läuft normalerweise rund um die Uhr. Eine 100%ige Vollständigkeit kann aber selbstverständlich nicht garantiert werden da auch dieser Server ab und an zu Wartungszwecken mal neu gestartet werden muß - oder weil einfach ab und an mal Dinge schief laufen.
